Defining the project structure

Wireframing is the most important step in the software development cycle. My teacher once said that "the more you delay coding the faster you finish development".

My initial commit contained empty files and folders that acted as entries into which I could populate later once I ahd decided on how to go about the design
So empty html files for each route, assets, script and styles folder for jpegs, javascript and css files respectively.

Good code is readable code, which is why I chose to use the Prettier library for code formatting to maintain a coherent aesthetic for every file, making the development experience a tad bit less chaotic.

Templating & Consistency

Maintaining consistent design across multiple pages, particularly for the navigation can be a challenge. Every page needs to look and feel the same and hence redundacy is inevitable.

The simplest way of solving this problem with as little redundancy as possible is simply to establish your objective and solve the first piece before copy and paste to the rest of these files
For the nav issue, I had to solve the responsiveness issue for just the home page and then copy over the markup as a template to the rest of the files, giving it that consistent and intentional feel.

All the files share a global.css stylesheet to give them the same look and feel without much of a hustle, and each file has their own specifications in a separate stylesheet; creating a neat and convenient Separation of Concerns.
The global.css file defines a list of CSS variable that I used in each page to maintain that uniform aesthetic without unnecessary redundancy.

Markup Validity

It is important to ensure that your markup is valid and SEO friendly. Despite the fact that all markup is inherrently the same, and that you can use the humble div to represent a heading tag, or a span in place of an i tag and style it using CSS classes, it is still crucial to use the right tool for the job.
Valid markup doesn't only help search engines, it is a crucial tool in web accessibility, allowing users to locate different sections of your code neatly.

My markup conveniention is splitting essential sections of the code to their individual functions; nav then main then footer for instance.

It is crucial to add meta tags for the required metadata especially for viewport scaling, and finally, KeepItSimpleStudid. No one likes a bloated site so restrict your site to the minimum requirements
